# PROJ-0010 — Yolo Mode Implementation Plan

## Overview

Invert the default run mode of `adversarial-architect`: the tool now stops after every sprint so the user can review (and optionally edit) the generated files before the next sprint begins. `--yolo` is the opt-in escape hatch that preserves today's unattended behavior. The pause leans entirely on the existing `--resume` mechanism from PROJ-0009 — no new state is persisted and no new recovery logic is added.

## Current State Analysis

- `cli.py:32-76` — Typer options; `--strict` at line 68-75 is the exact pattern to mirror for a new boolean flag.
- `cli.py:169-183` — `asyncio.run(run_harness(..., strict=strict))` threads CLI flags down.
- `harness.py:271-281` — `run_harness(prd, output_dir, resume, config, context_files=None, codebase=None, *, strict=False)`.
- `harness.py:368` — outer sprint loop `for sprint in SPRINTS[start_sprint_idx:]:`. The iteration body ends at line 916 after `generate_sprint_documentation`.
- `harness.py:377-395` — **already-completed sprints on resume hit `continue` before reaching the body's end**, so a stop check placed at the bottom of the loop body is naturally only triggered for sprints processed *this invocation*.
- `harness.py:430-431` — `progress.current_sprint = sprint.number` + `save_progress` before the round loop; `progress.completed_sprints += 1` at line 883; `save_progress` again at 884. So state is always consistent at the stop point.
- `harness.py:918-923` — `run_final_agreement` runs after the loop. On resume after a sprint-7 stop, the outer loop body short-circuits via the "Already passed — skipping" path at line 378-382, and control falls through to final-agreement automatically — no special-casing needed.
- `sprints.py:15-84` — every `SprintDefinition` already carries `primary_files: list[str]` (e.g. `["c1-context.md"]`, `["decisions/"]`) — exactly what to surface as "files to review."
- `models/progress.py` — `HarnessProgress` is unchanged. Per the ticket, `--yolo` is a per-invocation flag and must not be persisted.

## Desired End State

```
# Default: sprint-by-sprint checkpoint mode
adversarial-architect --prd my.prd.md --output ./arch
  → completes sprint 1 → prints review files + resume command → exits 0
  $ adversarial-architect --prd my.prd.md --output ./arch --resume
  → completes sprint 2 → prints review files + resume command → exits 0
  ... (etc through sprint 7) ...
  $ adversarial-architect --prd my.prd.md --output ./arch --resume
  → runs final mutual-agreement round, writes index, completes.

# Yolo: unattended, today's behavior
adversarial-architect --prd my.prd.md --output ./arch --yolo
  → runs all 7 sprints + final agreement unattended.

# Resume + yolo: resume and run remaining sprints unattended
adversarial-architect --prd my.prd.md --output ./arch --resume --yolo
```

Verified by:

```bash
adversarial-architect --help        # shows --yolo flag
uv run ruff check deep_architect/ tests/ \
  && uv run mypy deep_architect/ \
  && uv run python -m pytest tests/ -v \
  && uv run bandit -r deep_architect/ -ll
```

### Key Discoveries

- Placing the pause at the end of the outer loop body (after line 916) means the `continue` at line 382 ("Already passed — skipping") automatically protects us from stopping after a sprint that was *not* processed this invocation — this is exactly the desired behavior on `--resume`.
- A sprint that ended with status `failed` triggers `return` at line 881 — that is already a hard stop. The new pause logic is only reached when a sprint ends in `passed` or `accepted`, which is exactly when human review is meaningful.
- `sprint.primary_files` is already the correct source for the "files to review" list. No new data model needed.
- The resume command the user needs is always a variant of the command they already typed — the simplest robust message is "re-run your command with `--resume` appended" plus the output directory as an anchor. We do *not* try to reconstruct `argv` from memory.

## What We're NOT Doing

- No new field on `HarnessProgress` or `SprintStatus`. `--yolo` is per-invocation only.
- No new checkpoint, no new abort/recovery logic, no re-runs of the critic after human edits (ticket: "trusts the human and moves directly to sprint N+1").
- No changes to `io/files.py`, `git_ops.py`, `exit_criteria.py`, `config.py`, `agents/*`, or prompt files.
- No changes to default behavior when `--yolo` is set — parity with today's flow.
- No gating of the final mutual-agreement round as a separate "sprint." It runs automatically on the next `--resume` after sprint 7 stops (via the existing empty-loop-fall-through path).
- No interactive "press enter to continue" inside a single invocation — the pause always means the process exits and the user re-invokes.

## Implementation Approach

Three small edits in two files plus one new test:

1. Add `yolo: bool` CLI option (mirror of `--strict`) and thread to `run_harness`.
2. Add `yolo: bool = False` kwarg to `run_harness`; at the end of each outer-loop iteration in non-yolo mode, print review info and `return`.
3. Add a unit test verifying flag pass-through and help text visibility.

## Phase 2 — Harness pause logic

### Overview

Accept `yolo` in `run_harness`; at the bottom of each outer-loop iteration, when `yolo` is False, emit a review message and return. The pause runs *after* `progress.completed_sprints += 1`, the sprint-boundary commit, and `generate_sprint_documentation` — so all state is persisted and committed before exit.

### Changes Required

#### 1. `deep_architect/harness.py` — signature

**File**: `deep_architect/harness.py`

At line 271-281, extend `run_harness`:

```python
async def run_harness(
    prd: Path | None,
    output_dir: Path,
    resume: bool,
    config: HarnessConfig,
    context_files: list[Path] | None = None,
    codebase: Path | None = None,
    *,
    strict: bool = False,

) -> None:
```

#### 2. `deep_architect/harness.py` — pause helper

**File**: `deep_architect/harness.py`

Add a small module-level helper near the other top-level helpers (e.g. after `_log_resume_scores` at line 268):

```python
def _print_sprint_pause(
    sprint: SprintDefinition,
    output_dir: Path,
    progress: HarnessProgress,
) -> None:
    """Print the review checkpoint message shown between sprints in non-yolo mode."""
    files_label = ", ".join(sprint.primary_files) if sprint.primary_files else "(none)"
    next_sprint = sprint.number + 1
    more_to_go = next_sprint <= progress.total_sprints
    console.print()
    console.print(
        f"[green]Sprint {sprint.number}/{progress.total_sprints} complete:[/green] "
        f"{sprint.name}"
    )
    console.print(f"  Review files in [bold]{output_dir}[/bold]:")
    console.print(f"    {files_label}")
    if more_to_go:
        console.print(
            f"  When ready, re-run your command with [bold]--resume[/bold] "
            f"to continue with sprint {next_sprint}."
        )
        console.print(
            "  Add [bold]--yolo[/bold] to run the remaining sprints unattended."
        )
    else:
        console.print(
            "  All 7 sprints complete. Re-run your command with [bold]--resume[/bold] "
            "to run the final mutual-agreement round."
        )
    console.print()
    logger.info(
        "[Sprint %d] Paused for human review (non-yolo mode). "
        "Re-run with --resume to continue.",
        sprint.number,
    )
```

#### 3. `deep_architect/harness.py` — insert the pause

**File**: `deep_architect/harness.py`

At line 914-916 (end of the outer loop body, right after the sprint-documentation `try/except`), insert the pause just before the loop iterates:

```python
            logger.info(f"[Sprint {sprint.number}] Documentation generated: {sprint_doc_path.name}")
        except Exception as e:
            logger.warning(f"[Sprint {sprint.number}] Failed to generate sprint documentation: {e}")

        if not yolo:
            _print_sprint_pause(sprint, output_dir, progress)
            return
```

Rationale for the exact placement:

- `progress.completed_sprints += 1` (line 883) and `save_progress` (line 884) have already run — state on disk reflects the completed sprint.
- The sprint-boundary `git_commit` (line 886-891) has already captured the feedback/progress files.
- `generate_sprint_documentation` (line 905-914) has already written the per-sprint doc.
- The `continue` path at line 382 (already-completed sprints on resume) does NOT reach this code, so we correctly *do not* pause after skipped sprints.
- Sprints that end with status `failed` return at line 881 before ever reaching this code — the hard-stop semantics for failures are preserved.

```python
"""CLI-level tests for the --yolo flag wiring."""
from __future__ import annotations

from unittest.mock import patch

from typer.testing import CliRunner

from deep_architect.cli import app


runner = CliRunner()


def test_help_text_mentions_yolo() -> None:
    result = runner.invoke(app, ["--help"])
    assert result.exit_code == 0
    assert "--yolo" in result.output


def test_yolo_flag_threaded_to_harness(tmp_path):  # type: ignore[no-untyped-def]
    prd = tmp_path / "prd.md"
    prd.write_text("# Stub PRD\n")
    out = tmp_path / "arch"
    out.mkdir()

    # Patch validate_git_repo so we do not need a real repo, and patch
    # run_harness so the test does not actually run the expensive loop.
    with patch("deep_architect.cli.validate_git_repo"), \
         patch("deep_architect.cli.asyncio.run") as mock_run:
        result = runner.invoke(
            app,
            ["--prd", str(prd), "--output", str(out), "--yolo"],
        )

    assert result.exit_code == 0, result.output
    # The coroutine passed to asyncio.run() is the result of run_harness(...);
    # inspect its bound kwargs via the unittest mock call args.
    assert mock_run.called
    coro = mock_run.call_args.args[0]
    # The coroutine's cr_frame.f_locals holds the kwargs at suspension.
    assert coro.cr_frame.f_locals.get("yolo") is True
    coro.close()  # avoid "coroutine was never awaited" RuntimeWarning


def test_yolo_defaults_false(tmp_path):  # type: ignore[no-untyped-def]
    prd = tmp_path / "prd.md"
    prd.write_text("# Stub PRD\n")
    out = tmp_path / "arch"
    out.mkdir()

    with patch("deep_architect.cli.validate_git_repo"), \
         patch("deep_architect.cli.asyncio.run") as mock_run:
        result = runner.invoke(app, ["--prd", str(prd), "--output", str(out)])

    assert result.exit_code == 0, result.output
    coro = mock_run.call_args.args[0]
    assert coro.cr_frame.f_locals.get("yolo") is False
    coro.close()
```

Notes:
- The `coro.cr_frame.f_locals` inspection is the same pattern used to verify kwarg pass-through without awaiting. If mypy/ruff flag the dynamic access, add a targeted `# type: ignore[union-attr]` on that line only.
- The test does NOT exercise `run_harness` internals — those are covered by the manual verifications and existing tests.

```markdown
### Unattended vs sprint-by-sprint mode

By default, `adversarial-architect` stops after each of the 7 sprints, prints the
```

## Testing Strategy

### Unit Tests
- `test_cli_yolo.py::test_help_text_mentions_yolo` — `--yolo` in help output.
- `test_cli_yolo.py::test_yolo_flag_threaded_to_harness` — CLI passes `yolo=True` down when `--yolo` is set.
- `test_cli_yolo.py::test_yolo_defaults_false` — without the flag, `yolo=False` propagates.
- All existing tests must continue to pass unchanged (the new kwarg has a default).

### Integration / Manual Tests
Covered in the Phase 2 Manual Verification block. Key paths:
1. Default → stop after sprint 1 → resume → sprint 2 → etc.
2. `--yolo` → full unattended run.
3. `--resume` (no `--yolo`) → run one more sprint then stop.
4. `--resume --yolo` → run the rest unattended.
5. After sprint 7 stop → `--resume` triggers only the final mutual-agreement round.
6. Resume into a run with sprints already completed → no spurious pause messages for skipped sprints.
7. Sprint `failed` path → unchanged hard stop (no review message).

## Performance Considerations

None. The change is a single conditional `return` at the sprint boundary plus Rich console output.

## Migration Notes

Behavior change: existing users who relied on the unattended default must add `--yolo` to their invocation. This is the explicit ticket decision ("the default (non-yolo) behavior is the new standard; yolo is the opt-in escape hatch"). Document in `README.md` and in the `--yolo` help text as already specified.

No on-disk format changes — prior checkpoints continue to work.
